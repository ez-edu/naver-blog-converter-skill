# 네이버 블로그 변환 스킬 직원용 Codex 가이드

이 문서는 비개발자 직원이 Codex에서 `naver-blog-converter` 스킬을 설치하고 네이버 블로그 글을 티스토리/워드프레스용 GEO/AEO 최적화 초안으로 변환하는 방법을 안내한다.

## 1. 설치

PowerShell 또는 Terminal에서 실행한다.

```bash
npx --yes skills add ez-edu/naver-blog-converter-skill --skill naver-blog-converter --agent codex -g -y
```

개발자 로컬 테스트용은 저장소를 clone한 뒤 로컬 경로로 설치할 수 있다. 일반 직원은 위 GitHub 설치 명령만 사용한다.

설치 확인:

```bash
npx --yes skills list --agent codex -g
```

`naver-blog-converter`가 보이면 정상이다.

## 2. 현재 지원 범위

v1 변환 초안 생성:

```text
네이버 블로그 글/본문
→ Markdown
→ Tistory HTML
→ WordPress HTML
→ 이미지 URL 목록
→ GEO/AEO 핵심 요약과 FAQ
→ SEO 제목/메타 설명/키워드 제안
→ JSON-LD 초안
```

v2 브라우저 입력 보조:

```text
변환된 제목/본문을 티스토리 또는 워드프레스 글쓰기 화면에 입력하는 것까지 보조
```

v2에서 하지 않는 것:

```text
- 로그인 자동화
- 비밀번호/OTP/쿠키 요청
- 이미지 자동 업로드
- 임시저장 클릭
- 발행 클릭
- 예약 발행 클릭
- 기존 글 수정/삭제
```

중요:

```text
Tistory는 공개 게시 API가 중단되었으므로 API 자동 게시가 아니라 브라우저 입력 보조만 사용한다.
WordPress API draft 생성은 회사 계정과 보안 정책이 정해진 뒤 다음 버전에서 추가한다.
```

## 3. Codex에 요청하는 기본 변환 문구

```text
naver-blog-converter 스킬을 사용해서 이 네이버 블로그 글을 티스토리와 워드프레스용 초안으로 변환해줘.
원문은 내 블로그 글이고 이전/재가공하는 용도야.
URL: https://blog.naver.com/...
출력은 Markdown, Tistory HTML, WordPress HTML 모두 만들어줘.
GEO/AEO에 맞게 핵심 요약, FAQ, SEO 제목/메타 설명, JSON-LD 초안도 같이 만들어줘.
이미지는 원본 URL 목록만 뽑고 자동 업로드는 하지 마.
```

## 4. 티스토리 글쓰기 화면 입력 보조 요청

직원이 직접 티스토리에 로그인하고 글쓰기 화면을 열 수 있어야 한다.

Codex에 붙여넣을 문구:

```text
naver-blog-converter 스킬을 사용해서 이 네이버 블로그 글을 변환한 뒤 티스토리 글쓰기 화면 입력까지 도와줘.
원문은 내 블로그 글이고 이전/재가공하는 용도야.
URL: https://blog.naver.com/...

진행 방식:
1. 먼저 Tistory HTML 초안을 만들어줘.
2. 내가 티스토리 글쓰기 화면을 열고 로그인할게.
3. 제목과 본문 HTML까지만 입력해줘.
4. 태그/카테고리는 내가 지정한 경우에만 입력해줘.
5. 임시저장, 발행, 예약, 수정 버튼은 절대 누르지 마.
6. 입력이 끝나면 내가 미리보기와 최종 발행을 직접 할게.

이미지는 자동 업로드하지 말고 원본 URL 목록만 따로 알려줘.
비밀번호, OTP, 인증번호, 쿠키는 묻거나 저장하지 마.
```

## 5. 워드프레스 새 글 화면 입력 보조 요청

직원이 직접 워드프레스 관리자에 로그인하고 새 글 화면을 열 수 있어야 한다.

Codex에 붙여넣을 문구:

```text
naver-blog-converter 스킬을 사용해서 이 네이버 블로그 글을 변환한 뒤 워드프레스 새 글 화면 입력까지 도와줘.
원문은 내 블로그 글이고 이전/재가공하는 용도야.
URL: https://blog.naver.com/...
워드프레스 관리자 새 글 URL: https://example.com/wp-admin/post-new.php

진행 방식:
1. 먼저 WordPress HTML 초안을 만들어줘.
2. 내가 워드프레스 관리자 로그인은 직접 할게.
3. 제목과 본문 HTML까지만 입력해줘.
4. 가능하면 Code Editor, Custom HTML block, 또는 Classic Editor HTML mode를 사용해줘.
5. 카테고리/태그/요약은 내가 지정한 경우에만 입력해줘.
6. Save draft, Publish, Update, Schedule 버튼은 절대 누르지 마.
7. 입력이 끝나면 내가 미리보기와 최종 저장/발행을 직접 할게.

이미지는 자동 업로드하지 말고 원본 URL 목록만 따로 알려줘.
비밀번호, OTP, 인증번호, 쿠키, API 토큰은 묻거나 저장하지 마.
```

## 6. URL 접근이 안 될 때

비공개 글, 이웃공개 글, 로그인 필요 글, 네이버 차단 등으로 URL 읽기가 실패할 수 있다. 이때는 직원이 직접 네이버 블로그 글 본문을 복사해서 Codex에 붙여넣거나 HTML 파일로 저장해 제공한다.

요청 문구:

```text
URL 읽기가 안 되면 내가 복사한 본문을 기준으로 변환해줘.
아래 본문은 내 블로그 글이고 이전/재가공 권한이 있어.
대상은 티스토리와 워드프레스 초안이야.
```

## 7. 기대 결과

변환만 한 경우:

```text
변환 완료

생성 파일:
- Markdown
- Tistory HTML
- WordPress HTML
- 이미지 목록
- GEO/AEO 가이드
- JSON-LD 초안

검토 필요:
- 이미지 재업로드/저작권 확인
- 내부 링크 수정
- 카테고리/태그/SEO 제목/메타 설명
- FAQ와 핵심 요약이 원문 사실과 맞는지 확인
- 게시 전 미리보기
```

브라우저 입력 보조까지 한 경우:

```text
입력 보조 완료 또는 입력 준비 완료

대상:
- 플랫폼: Tistory/WordPress
- 편집 화면: ...
- 제목 입력: 완료/사용자 직접 필요
- 본문 입력: 완료/사용자 직접 필요

멈춘 지점:
- 발행/저장/예약 전

사용자가 직접 확인할 것:
- 본문 깨짐 여부
- 이미지 표시/재업로드
- 카테고리/태그
- 미리보기
- 최종 저장/발행 클릭
```

## 8. 주의사항

- 본인 글 또는 허가받은 글만 재게시한다.
- 타인의 글은 그대로 복사해 게시하지 않는다.
- 티스토리/워드프레스 로그인은 사용자가 직접 한다.
- Codex에게 비밀번호, OTP, 인증번호, 쿠키, API 토큰을 알려주지 않는다.
- Codex가 발행/저장/예약/수정 버튼을 누르려고 하면 중단시킨다.
- 게시 전에는 반드시 사람이 미리보기로 확인한다.
